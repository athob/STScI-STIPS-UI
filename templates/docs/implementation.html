STIPS Implementation

STIPS is implemented in a Python module (itself named "stips"). The module has the following requirements and components:

Requirements

  • Python Modules
    • astropy: WCS operations, FITS file input/output, convolution, tables, cosmology
    • automodinit: this is required in order for the instruments sub-module to find and import all available instruments without hard-coding which instruments are present.
    • copy
    • esutil: turning database retrievals into numpy arrays
    • glob
    • importlib
    • inspect
    • logging
    • math
    • montage_wrapper: required for creating mosaics of multi-detector images and dither patterns
    • numpy: required for arrays and array operations.
    • os
    • pandeia: required for JWST/WFIRST throughputs, background countrates
    • pysynphot: required for simulating observations and handling source spectra and bandpasses
    • pytest: testing interface
    • scipy: convolution, grid interpolation, image operations
    • shutil
    • sqlite3: stellar population information is stored in an sqlite database
    • sys
    • tempfile
    • time
    • uuid
    • webbpsf: required for generating JWST and WFIRST instrumental PSF functions
  • External Programs
    • Montage (creating image mosaics)
  • External Data
    • CDBS reference data
    • Pandeia reference data for all instruments you wish to simulate
    • Webbpsf reference data for all instruments you wish to simulate
  • Reference Data STIPS requires a reference data directory which it looks for via the environment variable ‘stips_data’. This data must be present for STIPS to function correctly. Currently, it may be obtained directly from Brian York, or from the STSCI-STIPS-UI github package.

Scene Generation

STIPS is currently able to generate two type of scenes, stellar populations and background galaxies.

Stellar Populations

Stellar populations are generated by the StarGenerator class in the STIPS stellar_module. The stellar module creates a population given

  • Population age (years)
  • Population metallicity ([Fe/H])
  • IMF (one of Power Law, Kroupa, Salpeter, Chabrier, Schechter, or Schechter with exponential cut-off).Â 

The module also uses a database of isochrones derived from CMD 2.7 and, for determining count rates, a saved grid of count rates as determined by the phoenix grid.
