STIPS Release Notes

Version 1.0.0rc3 (July 5, 2017)

  • Updated STIPS to extend Sersic profiles out until the flux drops below the noise floor.
  • Updated STIPS documentation to indicate that Sersic profiles have an average surface brightness rather than a flux.
  • Updated internal STIPS catalogue to treat half-light radius as being in instrument-native pixels.
  • Fixed a number of bugs

Version 1.0.0rc2 (June 23, 2017)

  • Updated STIPS to use Pandeia Engine 1.1, and Pandeia data 1.1
  • Updated Sersic Profile generation to extend to 100x the effective radius, rather than the full detector frame
  • Fixed a bug where Sersic profile flux was applied as surface brightness
  • Added the ability to directly apply in-memory tables to STIPS via the python module
  • Fixed a bug where Sersic profile generation didn't properly update the observed catalogue
  • Fixed a number of bugs

Version 1.0.0rc1 (June 6, 2017)

  • Updated Sersic profile generation to use Astropy Sersic2D
  • Separated STIPS package from STIPS web UI, and moved both to github
  • Fixed several bugs

Version 0.6.3

June 21, 2016

  • Updated webbpsf to 0.5.0
  • Updated pandeia to 0.1.dev0

Version 0.6.2

June 21, 2016

  • Fixed a bug where final co-ordinate values weren't renumbered to deal with oversampling or PSF buffer.

Version 0.6.1

June 14, 2016

  • All internal IPAC tables now use lowercase column names
  • Fixed a bug where STIPS would crash if no binary stars were present in a chunk
  • Fixed a bug where conflicting variable names would limit star lists to 100,000 stars
  • Fixed a bug where tables larger than the chunk size could have variable column width
  • Made column format consistent in initial outputs.

Version 0.6.0

May 18, 2016

  • Initial Public Version
